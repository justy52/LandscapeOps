# LandscapeOps Docs

This documentation set defines the Phase 0 product and engineering foundation for LandscapeOps.

## Product

- `01-product/product-spec.md` defines the target customer, value proposition, workflows, and non-goals.
- `01-product/app-scope.md` defines what belongs in Phase 0 and what is deferred.

## Architecture

- `02-architecture/tech-stack.md` records the chosen stack and integration responsibilities.
- `02-architecture/data-model.md` explains the tenant-owned domain model.
- `02-architecture/api-spec.md` sketches route and server action conventions.
- `02-architecture/auth-permissions.md` defines Clerk org and role direction.

## Features

The feature docs describe the initial shape for CRM, estimating, contracts, job lifecycle, scheduling, field operations, client portal, and reporting.

## Finance

Finance docs define invoicing, payment, margin, tax, discount, and audit guardrails. Any change here should receive extra review.

## Design

Design docs define the premium UI direction, colors, density, accessibility, and mobile-first expectations.

## Build, Security, Testing, Operations

The remaining docs define phased delivery, AI handoff rules, security/deploy/legal constraints, QA expectations, and settings/onboarding direction.
